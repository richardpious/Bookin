# File Organization Protocol

## Purpose
This protocol defines the mandatory directory structure and procedures for all simulation-related work within the agent workspace. The agent must follow this structure whenever creating, executing, analyzing, or storing simulation data, ensuring consistency and auditability.

---

## Directory Structure
All simulation-related work must be organized under the project's `../logs` directory (located at `/home/dell/Documents/claw/logs`).

```text
../logs/
└── session_<timestamp>/
    └── run_<n>/
```

### Session Rules
A session represents a single user interaction context. Each session must have its own directory using a unique timestamp to identify the session.
* Format: `session_YYYY_MM_DD_HH_MM`
* Example: `session_2026_06_05_11_45`

### Run Rules
Each simulation execution is a run. Runs must be stored sequentially as `run_1`, `run_2`, etc., inside the active session directory. A new simulation execution must create a new run directory unless otherwise requested by the user.

---

## Standard Procedures

### Before Simulation
1. Identify the active session directory under `logs/`. Create it if it does not exist.
2. Determine the next available `run_<n>` directory within the current session.
3. Create the `run_<n>` directory.
4. Change the working directory to this `run_<n>` folder.

### Simulation Execution
1. Simulations must be executed from inside the `run_<n>/` directory.
2. Ensure all generated log files (e.g., `stats.txt`, `watch_out.txt`) are saved directly into this folder.

### After Simulation
1. Ensure all relevant output files (e.g., `stats.txt`, `watch_out.txt`) generated by the simulation are properly stored in the current `run_<n>` directory.
2. Maintain a clear audit trail from config to execution to results.

### Configuration Management
- Base configurations: `/home/dell/Documents/claw/booksim/src/configs/`
- Custom configurations: `/home/dell/Documents/claw/agent/configs/custom/`
- Never modify base configurations directly.

---

## Prohibited Behavior
The agent MUST NOT:
* Run simulations from the project root.
* Run simulations from arbitrary directories.
* Store simulation outputs directly in `logs/` or a session directory without a run folder.
* Mix outputs from multiple runs in the same directory.
* Create simulation artifacts outside the required structure.

---

## Exception Handling
The only exception to this workflow is when the user explicitly instructs the agent to use a different directory structure or execution location. In the absence of explicit user instructions, this protocol is mandatory.

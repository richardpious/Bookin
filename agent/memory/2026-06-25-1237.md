### How OpenClaw Handles Reasoning/Thinking

There are **two separate mechanisms** at play:

1. **Thinking level** (`/think` or `/t`): Controls how much internal reasoning budget the model uses *during* generation. This is model-specific — some models think more deeply when given a higher budget.

2. **Reasoning visibility** (`/reasoning`): Controls whether OpenClaw *displays* the thinking/reasoning blocks to you as a **separate message** prefixed with `Thinking`.

### How Models Differ

Different providers emit reasoning in very different ways:

| Provider | Behavior |
|---|---|
| **Anthropic Claude** | Emits structured `thinking` blocks as distinct content — OpenClaw can extract and display these cleanly |
| **DeepSeek V4** | Emits `reasoning_content` in the stream — OpenClaw can render this separately |
| **Google Gemini** | Uses **internal thinking budgets** (`thinkingBudget` / `thinkingLevel`) — the thinking happens inside the model but isn't emitted as separate, extractable blocks |
| **OpenAI** | Uses `reasoning.effort` parameter — reasoning is internal to the model response |

import { getSessionEntry } from "@openclaw/plugin-sdk/session-store-runtime";

function getThinkingLevels(agentId: string, sessionKey: string) {

### 2. Policy Resolution Logic (The Engine)
The core logic for thinking levels lives in the `plugin-sdk` and `runtime` layers. It doesn't rely on hardcoded maps in the UI; it dynamically asks the runtime what a model supports.

*   **`resolveThinkingPolicy({ provider, model })`**: This is the most critical function. Setting a thinking level is a state mutation, not a message directive.

1.  **Trigger**: When a user selects a level in the UI, the frontend calls `sessions.patch` (or the SDK's `patchSessionEntry`).

When this parameter is set to a non-empty string, BookSim will open a file with that name and, for each batch of packets injected by the `BatchTrafficManager`, write a line containing the number of packets sent by each source node during that batch.

### What "batch only" means
The `BatchTrafficManager` operates in batches: it injects packets in groups (each batch consists of `_batch_size` packets per source node, unless injection is throttled). At the start of each batch, it resets a per‑source packet counter (`_packet_seq_no[source]`) to zero. As packets are injected, the counter for the corresponding source is incremented. When all sources have finished sending their batch (i.e., every source has sent `_batch_size` packets or injection has stopped), the manager writes the current values of `_packet_seq_no` (one integer per source) to the file specified by `sent_packets_out`, followed by a newline. This process repeats for each batch until `_batch_count` batches have been sent.

Thus, the output is "batch only" in the sense that it produces **one line per batch**, summarizing the per‑source packet counts for that batch only—not cumulative totals or per‑packet details.

If `sent_packets_out` is left empty (the default), no file is created and no such batch‑wise packet‑count output is generated.

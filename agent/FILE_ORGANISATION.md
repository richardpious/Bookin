# File Organization

## Purpose

This skill defines the mandatory directory structure for all simulation runs and generated artifacts.

The agent MUST follow this structure whenever creating, executing, analyzing, or storing simulation data unless the user explicitly requests a different location.

---

## Required Directory Structure

All simulation-related work must be organized under the project's `../logs` directory.

```text
logs/
└── session_<timestamp>/
    └── run_<n>/
```

Example:

```text
logs/
└── session_20260605_114530/
    ├── run_1/
    ├── run_2/
    └── run_3/
```

---

## Session Rules

A session represents a single user interaction context.

Each session must have its own directory:

```text
logs/session_<timestamp>/
```

The timestamp should uniquely identify the session.

Examples:

```text
session_2026_06_05_11_45
session_2026_06_05_15_22
```

All runs belonging to a session must be stored within that session directory.

---

## Run Rules

Each simulation execution is considered a run.

Runs must be stored as:

```text
run_1
run_2
run_3
...
```

inside the active session directory.

Example:

```text
logs/
└── session_20260605_114530/
    ├── run_1/
    ├── run_2/
    └── run_3/
```

A new simulation execution must create a new run directory unless the user explicitly requests reuse of an existing run.

---

## Simulation Execution Rules

Before executing a simulation, the agent must:

1. Identify the active session.
2. Create a session directory if one does not already exist.
3. Determine the next available run number.
4. Create the corresponding `run_<n>` directory.
5. Change the working directory to that run directory.
6. Execute the simulation from within that run directory.

The simulation workspace must be the run directory.

Example:

```text
logs/session_2026_06_05_11_45/run_4/
```

---

## Storage Rules

All files generated by a simulation should be stored inside the corresponding run directory.

Examples include:

* Configuration files
* Simulation logs
* Output files
* Analysis artifacts
* Generated reports
* Intermediate files

No simulation artifacts should be written outside the active run directory unless explicitly required.

---

## Prohibited Behavior

The agent MUST NOT:

* Run simulations from the project root.
* Run simulations from arbitrary directories.
* Store simulation outputs directly in `../logs/`.
* Store simulation outputs directly in a session directory.
* Mix outputs from multiple runs in the same directory.
* Create simulation artifacts outside the required structure.

The following are invalid:

```text
project_root/output.log
project_root/booksim.log
logs/output.log
logs/session_2026_06_05_11_45/output.log
```

---

## Exception Handling

The only exception to this workflow is when the user explicitly instructs the agent to use a different directory structure or execution location.

In the absence of explicit user instructions, this skill is mandatory and takes precedence over convenience.
